FCAN Get WMB Block

This block allows the user to get the wake-up message buffer.

Block Image

Inputs:

  • None

Outputs:

  • ID (uint32)
  • Data (uint8(8))
  • Length (uint8)

Parameters and Dialog Box

CAN Bus

Selects the CAN Bus.

Wake up Message Buffer

There are four WMB's (Wake up Message Buffers) used to store incoming messages in Pretended Networking mode. Up to four messages can be stored.

When Needed matches value is 1 or Enable multiple matches is unchecked in FCAN_Config block, just one message is received if matching the filtering criteria, and this message is stored in WMB0. If Needed matches value is between 2 and 4, WMB1, WMB2 and WMB3 are used to store the second, third and fourth matching messages, respectively. If Needed matches value is greater than 4, the last four matching messages are stored in the WMBs, respecting the WMB index to indicate the arrival order, the latest is stored in WMB3.

Only the valid data bytes of the incoming match message is stored in data field of WMBs. The non-valid data bytes is read as zero. In case of DLC=0 and RTR=1 the data field is filled with zero. In any of the above cases, the wake up interrupt is generated just when the filtering criteria is completed
